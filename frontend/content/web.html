Frontend Handbook — HTML • CSS • JavaScript

Format: one topic per block (Syntax • Example • Notes). Static, read-only, mobile-friendly, and print-ready. No external links or scripts.

HTML — Document & Structure

Document skeleton
HTML layout of a basic page
Syntax
<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Title</title>
</head>
<body>...</body>
</html>
NotesAlways include responsive viewport. Use a single, descriptive <title>.
Semantic layout: header • nav • main • section • article • aside • footer
HTML accessible structure
Syntax
<header>Brand</header>
<nav>...</nav>
<main>
  <section><article>Content</article></section>
  <aside>Related</aside>
</main>
<footer>©</footer>
NotesOne <main> per page. Use headings in logical order (h1→h6).
Headings & paragraphs
HTML h1—h6 • p • small • strong • em
Syntax<h1>Title</h1> ... <h6> • <p>Text</p> • <strong> • <em>
Example
<h1>Product</h1>
<p>A short <strong>bold</strong> point with <em>emphasis</em>.</p>
NotesUse semantic emphasis (<strong>/<em>), not just CSS.

HTML — Links, Media, Lists, Tables

Anchors & in-page links
HTML <a> with href, fragment ids
Syntax<a href="#section1">Jump</a> • <h2 id="section1">...</h2>
Example
<a href="#details">Details</a>
<section id="details">...</section>
NotesUse meaningful text; avoid “click here”.
Images
HTML <img> with alt, width, height
Syntax<img src="image.jpg" alt="description" width="600" height="400">
NotesProvide accurate alt. Set intrinsic width/height to reduce layout shift.
Video & audio
HTML <video> • <audio> • <source>
Syntax
<video controls width="640">
  <source src="clip.mp4" type="video/mp4">
  Your browser does not support video.
</video>
NotesUse captions/subtitles for accessibility when available.
Lists
HTML ul • ol • dl
Syntax
<ul><li>Item</li></ul>
<ol><li>Step 1</li></ol>
<dl><dt>Term</dt><dd>Definition</dd></dl>
NotesUse dl for term–definition pairs.
Tables
HTML table • thead • tbody • tfoot • th • td • caption
Syntax
<table>
  <caption>Scores</caption>
  <thead><tr><th scope="col">Name</th><th scope="col">Score</th></tr></thead>
  <tbody><tr><th scope="row">Ana</th><td>92</td></tr></tbody>
</table>
NotesUse scope on th for screen readers.

HTML — Forms & Controls

Form & label basics
HTML form • label • input • button
Syntax
<form method="post">
  <label for="email">Email</label>
  <input id="email" type="email" required>
  <button type="submit">Send</button>
</form>
NotesAssociate labels with inputs using for/id. Provide name for form submission.
Common input types
HTML text • email • number • password • date • time • checkbox • radio • range • file • color • tel • url • search
Syntax
<input type="text">
<input type="number" min="0" step="1">
<input type="date">
<input type="checkbox">
<input type="radio" name="opt">
<input type="file" accept="image/*">
NotesUse constraints: required, min, max, pattern, maxlength, autocomplete.
Textarea, select, datalist
HTML multi-line & choices
Syntax
<textarea rows="4"></textarea>
<select><option>A</option><option>B</option></select>
<input list="cities">
<datalist id="cities"><option value="Delhi"></datalist>
NotesUse multiple on select for multi-select lists.

HTML — Embeds & Graphics

iframe
HTML embed another page/app
Syntax<iframe src="page.html" title="Embedded content" width="600" height="400"></iframe>
NotesAlways include a descriptive title for accessibility.
canvas & svg (basics)
HTML drawing APIs
Syntax
<canvas width="300" height="150"></canvas>
<svg viewBox="0 0 100 100"><circle cx="50" cy="50" r="40"/></svg>
Notescanvas is bitmap via JS; svg is vector DOM.

CSS — Selectors

Basic & combinators
CSS type • class • id • descendant • child • siblings
Syntax.card • #hero • h1, h2 • .nav > li • h2 + p • h2 ~ p
NotesUse classes for reuse; limit ID usage in CSS.
Attribute selectors
CSS powerful filtering
Syntax[disabled] • input[type="email"] • a[href^="#"] • a[href$=".pdf"] • [data-id*="42"]
Notes^ starts, $ ends, * contains.
Pseudo-classes & pseudo-elements
CSS states & generated content
Syntax:hover • :focus • :focus-visible • :disabled • :not() • :nth-child() • ::before • ::after
Example
.btn:hover { transform: translateY(-1px); }
.badge::before { content:"• "; }
NotesUse :focus-visible to show outlines only for keyboard users.

CSS — Box Model & Sizing

Padding • Margin • Border • Box-sizing
CSS spacing inside vs outside
Syntaxpadding: t r b l; • margin: • border:1px solid #ddd; • box-sizing:border-box;
Example
.panel{ padding:12px 16px; margin:12px 0; border:1px solid #e5e7eb; box-sizing:border-box; }
NotesInside=padding, outside=margin. Use box-sizing:border-box globally.
Width/height & min/max
CSS constraints
Syntaxwidth:100%; • max-width:1000px; • min-height:48px;
NotesImages: max-width:100%; height:auto; for responsive media.

CSS — Layout

Display & position
CSS block • inline • inline-block • flex • grid • sticky • absolute
Syntaxdisplay:flex|grid • position:relative|absolute|sticky • top/right/bottom/left
Example
.sticky { position: sticky; top: 0; background:#fff; }
Notesposition:relative establishes a containing block for absolutely positioned children.
Flexbox
CSS one-dimensional layout
Syntaxdisplay:flex; • gap: • justify-content: • align-items: • flex-wrap: • flex: grow shrink basis
Example
.row{ display:flex; gap:12px; align-items:center; justify-content:space-between; }
NotesUse flex:1 to make a child grow; or flex:0 0 240px for fixed basis.
CSS Grid
CSS two-dimensional layout
Syntaxdisplay:grid; • grid-template-columns: • grid-template-rows: • gap: • grid-area:
Example
.grid{ display:grid; grid-template-columns:repeat(3,1fr); gap:14px; }
NotesUse minmax(), auto-fit/auto-fill for responsive tracks.
Float & clear (legacy)
CSS old layout technique
Syntaxfloat:left|right; • clear:both;
NotesPrefer Flex/Grid; use float for text wrap with images.

CSS — Typography & Visuals

Fonts & text
CSS font-family • size • weight • line-height • letter-spacing • text-align • text-transform • text-overflow
Syntax
h1{ font-family:system-ui,Arial; font-size:24px; line-height:1.3; }
p{ text-align:justify; text-justify:auto; }
NotesUse unitless line-height for better inheritance.
Color & background
CSS color • background-color • gradients • images
Syntax
.btn{ color:#fff; background:#2563eb; }
.card{ background-image:linear-gradient(#fff,#f7f7fb); }
NotesUse accessible contrast; prefer HSL/RGB for opacity control.
Borders, radius, shadows, outline
CSS shape & depth & focus
Syntax
.card{ border:1px solid #e5e7eb; border-radius:12px; box-shadow:0 6px 20px rgba(0,0,0,.08); }
button:focus-visible{ outline:2px solid #0ea5e9; outline-offset:2px; }
NotesUse subtle shadows; never remove focus outlines without a replacement.

CSS — Effects & Responsive

Transforms & transitions
CSS motion & interaction
Syntax
.btn{ transition: transform .15s ease; }
.btn:hover{ transform: translateY(-1px) scale(1.01); }
NotesAnimate transform/opacity for performance. Avoid layout-thrashing properties.
@keyframes & animation
CSS reusable motion
Syntax
@keyframes fade { from{opacity:0} to{opacity:1} }
.notice{ animation: fade .4s ease; }
NotesPrefer short, subtle animations; respect reduced-motion preferences if needed.
@media (media queries)
CSS responsive design
Syntax
@media (max-width: 640px){
  .grid{ grid-template-columns:1fr; }
}
NotesMobile-first: write base styles, then add min-width breakpoints.
@supports (feature queries)
CSS progressive enhancement
Syntax
@supports (backdrop-filter: blur(4px)) {
  .glass{ backdrop-filter: blur(4px); }
}
NotesFallbacks go outside the block; enhanced styles inside.
@container (container queries)
CSS component-level responsiveness
Syntax
.card{ container-type: inline-size; }
@container (min-width: 480px){
  .card .meta{ display:flex; }
}
NotesEnable with container-type on a parent to query its size.

JavaScript — Language Essentials

Declarations & types
JS let • const • var • primitives & objects
Syntax
const n = 42; let msg = "hi"; let ok = true; let nothing = null; let u;
const obj = {a:1}, arr = [1,2];
NotesPrefer const for values you don’t reassign. Primitives are immutable; objects/arrays are mutable.
Functions & arrow functions
JS declarations & expressions
Syntax
function add(a,b){ return a+b; }
const mul = (a,b) => a*b;
NotesArrows don’t have their own this, arguments, or prototype.
Destructuring & spread/rest
JS concise assignment
Syntax
const {x,y} = point; const [a,b,...rest] = arr;
const copy = {...obj, k:2}; const merged = [...a1, ...a2];
NotesShallow operations — nested objects aren’t cloned.
Classes
JS syntactic sugar over prototypes
Syntax
class Point{ constructor(x,y){ this.x=x; this.y=y; } get r(){ return Math.hypot(this.x,this.y); } }
NotesUse getters/setters for computed properties and validation.

JavaScript — Arrays, Strings, Objects

Arrays — core methods
JS transform, search, slice vs splice
Syntax
[1,2,3].map(x => x*2)
[1,2,3].filter(x => x % 2)
[1,2,3].reduce((s,x) => s+x, 0)
[1,2,3].find(x => x===2)
arr.slice(1,3)  // copy
arr.splice(1,1) // mutate
Notesincludes, every, some, flat, flatMap are also useful.
Strings — handy methods
JS includes • startsWith • replaceAll • padStart/End • split/join
Syntax
"Hello".toLowerCase()
"Hi!".padStart(5,"_")
"abcabc".replaceAll("a","A")
"1,2,3".split(",").join("-")
NotesUse template literals: `Hello ${name}` for formatting.
Objects & JSON
JS Object.* • JSON.parse/stringify
Syntax
Object.keys(o), Object.values(o), Object.entries(o)
JSON.stringify(o), JSON.parse(str)
NotesBeware of circular structures when stringifying.
Map • Set • WeakMap • WeakSet
JS collections beyond objects
Syntax
const m = new Map([["a",1]]);
const s = new Set([1,2,2]); // {1,2}
NotesMap preserves insertion order and accepts any key type.

JavaScript — Dates, Numbers, Math

Date basics
JS creation & parts
Syntax
const d = new Date();
d.getFullYear(); d.getMonth(); d.getDate();
NotesMonths are 0-indexed. Avoid manual timezone math where possible.
Number & Math
JS parsing & utilities
Syntax
Number.isFinite(n), Number.isNaN(x)
parseInt("08",10), parseFloat("3.14")
Math.round, floor, ceil, max, min, random
NotesUse radix with parseInt. Math.random() is 0≤x<1.

JavaScript — Async & Storage

Promises & async/await
JS flow control for async tasks
Syntax
async function load(){
  try{
    const res = await fetch("/api");
    return await res.json();
  }catch(e){ /* handle */ }
}
NotesPromise helpers: Promise.all, allSettled, race, any.
Fetch API (JSON)
Web HTTP requests
Syntax
/* async */ fetch("/data.json").then(r => r.json())
NotesAlways check response.ok for non-2xx errors.
localStorage / sessionStorage
Web simple key–value storage
Syntax
localStorage.setItem("k","v");
const v = localStorage.getItem("k");
NotesOnly strings are stored; JSON-encode objects.

JavaScript — DOM

Select & traverse
DOM querySelector • closest • children • parentElement
Syntax
const el = document.querySelector(".btn");
el.closest(".card"); el.parentElement; el.children;
NotesquerySelectorAll returns a static NodeList (use spread to array).
Modify content & classes
DOM textContent • innerHTML • classList • style
Syntax
el.textContent = "Hello";
el.classList.add("active"); el.style.paddingTop = "12px";
NotesPrefer textContent over innerHTML to avoid injection.
Create & insert
DOM createElement • append • before/after
Syntax
const li = document.createElement("li");
li.textContent = "Item"; list.append(li);
NotesFor multiple inserts, use a DocumentFragment to reduce reflows.
Events & delegation
DOM addEventListener • target vs currentTarget
Syntax
list.addEventListener("click", (e) => {
  const btn = e.target.closest("button.remove");
  if(!btn) return;
  // handle remove...
});
NotesUse delegation for dynamic lists. Prevent default via e.preventDefault().

JavaScript — URLs & Regex

URL & URLSearchParams
Web parse & build URLs
Syntax
const u = new URL("https://example.com?a=1");
u.searchParams.get("a"); // "1"
u.searchParams.set("b","2");
NotesWorks well with relative URLs when given a base.
Regular expressions
JS test • match • replace
Syntax
/^\w+@\w+\.\w+$/i.test("a@b.com")
"abc123".replace(/\d/g,"") // "abc"
NotesEscape special chars; prefer raw strings for patterns containing backslashes.
